                  Michael Pamela Wood Browne • a year ago

                  The "var todo" statement basically creates a shorthand reference to an object so that you don't have to type out this.todos[position] every time you want to reference it.

                  It's up to the developer to decide if they are going to be referencing an object frequently and therefore if they need a nice little short variable like 'todo' to reference their objects.

                  If you meant why don't we create a global var todo variable equal to this.todos[position], this won't work because the value of 'position' only becomes obvious when the user asks the computer to take action on a particular todo. That is, we only find out the value of 'position' when the user asks the computer to either create, delete or update a specific todo.

                      Gordon Zhu Mod Michael • a year ago

                      Couldn't have said it better. The last thing I'd suggest is that if you're still unconvinced, try it out.

                      You'll quickly find that saving a todo variable that'll work throughout the whole object will be really awkward. That's because you'll have to change it every single time once you get the position argument from the user. This is more work than just getting this.todos[position].

                      In toggleCompleted, having a variable for todo just saves us some typing in line 23.

                      We can have 

                      todo.completed = !todo.completed ;

                      instead of 

                      this.todos[position].completed = !this.todos[position].completed;

                      .
